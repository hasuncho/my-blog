# CLAUDE.md

이 파일은 Claude Code가 이 프로젝트에서 작업할 때 참고하는 가이드입니다.

## 프로젝트 개요

마크다운(.md) 파일을 읽어 정적 블로그 웹사이트로 변환하는 프로젝트.
별도 서버나 빌드 도구 없이, 브라우저에서 마크다운을 파싱해 글 목록/본문을 렌더링하는 방식을 기본으로 한다.

## 기술 스택 / 제약

- **프레임워크 금지**: React, Vue, Next.js 등 프레임워크나 번들러(Webpack, Vite 등)를 사용하지 않는다.
- 순수 HTML, CSS, JavaScript(ES 모듈 또는 바닐라 스크립트)만 사용한다.
- 마크다운 파서가 필요하면 CDN 의존 대신 저장소에 직접 vendor 파일로 포함하거나, 간단한 파서를 직접 구현한다.
- Node.js는 로컬 개발 서버(정적 파일 서빙)나 빌드 스크립트가 필요할 때만 최소한으로 사용할 수 있다. 런타임 자체는 프레임워크 없는 순수 웹 기술로 유지한다.

## 디자인 원칙

- **깔끔하고 읽기 좋은 타이포그래피**: 본문 가독성을 최우선으로 한다 (적절한 line-height, 본문 폭 제한, 여백).
- **다크모드 지원**: `prefers-color-scheme` 미디어 쿼리를 기본으로 따르고, 사용자가 직접 전환할 수 있는 토글을 제공한다. 색상은 CSS 변수(커스텀 프로퍼티)로 관리해 라이트/다크 테마를 쉽게 전환한다.
- **모바일 반응형**: 모바일 우선으로 레이아웃을 설계하고, 작은 화면에서도 네비게이션과 본문이 깨지지 않도록 한다.
- 불필요한 장식이나 애니메이션보다 콘텐츠 자체가 돋보이는 미니멀한 디자인을 지향한다.

## 작업 시 유의사항

- 새로운 의존성(라이브러리, 프레임워크)을 추가하기 전에는 먼저 사용자와 확인한다.
- 컴포넌트화가 필요해도 프레임워크 없이 순수 JS로 구현한다 (예: 템플릿 문자열, `<template>` 태그, Web Components 등).
- 기능 구현 후에는 반드시 브라우저에서 실제로 라이트/다크 모드와 모바일 화면 크기를 확인한다.
