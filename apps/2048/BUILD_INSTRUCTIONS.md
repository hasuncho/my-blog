# Build 서브에이전트 지침 — 2048 게임

## 역할
너는 이 블로그 프로젝트(/Users/hasuncho/Desktop/my-blog)의 "Build" 단계를 담당하는 서브에이전트다.
`/apps/2048/spec.md`에 정리된 계획을 바탕으로 실제 2048 게임을 구현한다.

## 범위 제한 (반드시 준수)
- 수정/생성 가능한 파일은 **`/apps/2048/` 폴더 내부**로 한정한다.
- 블로그 본체 파일(`/index.html`, `/css/`, `/js/`, `/posts/`, `/package.json`, `/scripts/` 등)은
  **절대 건드리지 않는다.**
- `/apps/2048/spec.md`, `PLAN_INSTRUCTIONS.md`, `BUILD_INSTRUCTIONS.md`는 참고용이며 수정하지 않는다.

## 기술 제약
- React/Vue/Next.js 등 프레임워크, Webpack/Vite 등 번들러 사용 금지.
- 순수 HTML/CSS/바닐라 JS(ES 모듈 가능)만 사용.
- 외부 라이브러리/CDN을 새로 추가하지 않는다 (spec.md 기준으로도 불필요).

## 해야 할 일
1. `/apps/2048/spec.md`를 정독한다.
2. spec.md의 파일 구조에 따라 다음 파일을 작성한다:
   - `/apps/2048/index.html`
   - `/apps/2048/style.css`
   - `/apps/2048/game.js`
3. spec.md의 "8. 향후 Build 단계에서 참고할 구현 순서"를 따라 순서대로 구현한다.
4. 구현해야 할 핵심 기능:
   - 4x4 그리드, 방향키(↑↓←→)로 타일 이동/병합
   - 이동 후 랜덤 위치에 2 또는 4 타일 생성 (90%/10% 확률)
   - 현재 점수 + 최고 점수(localStorage) 점수판
   - 모바일 터치 스와이프 지원
   - 게임 오버 판정 및 오버레이, 새 게임 버튼
   - 2048 타일 등장 시 승리 판정 및 "계속하기" 옵션
   - 라이트/다크 모드 대응 (`prefers-color-scheme` 기반 CSS 변수)
   - 모바일 반응형 레이아웃
5. 구현이 끝나면 로컬에서 정적 파일로 열어도 동작하는지 (또는 가능하면 간단한 정적 서버로) 최소한의 자체 점검을 한다.

## 완료 후
무엇을 작성했는지, spec.md와 다르게 구현한 부분이 있다면 그 이유를 요약해서 답변한다.
Review 단계는 별도 서브에이전트가 담당하므로 네가 review.md를 작성할 필요는 없다.
