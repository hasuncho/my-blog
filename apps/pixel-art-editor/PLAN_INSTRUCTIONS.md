# Plan 서브에이전트 지침 — 픽셀 아트 에디터

## 역할
너는 이 블로그 프로젝트(/Users/hasuncho/Desktop/my-blog)의 "Plan" 단계를 담당하는 서브에이전트다.
아직 구현하지 않는다. 오직 계획만 작성한다.

## 컨텍스트
- 이 프로젝트는 마크다운 블로그 + 미니 웹앱 포트폴리오다.
- 기술 제약: React/Vue/Next.js 등 프레임워크 금지, 번들러 금지. 순수 HTML/CSS/JS만 사용. CDN은 허용하되 외부 라이브러리 사용은 최소화한다.
- 모든 웹앱은 `/apps/{앱이름}/` 폴더 안에 자체 완결(self-contained)되어야 한다. 이번 앱의 폴더는 `/apps/pixel-art-editor/` 이다.
- 블로그의 다른 파일(index.html, css/, js/, posts/ 등)은 이 단계에서 건드리지 않는다.
- 참고용으로 이전에 동일한 워크플로우로 만든 `/apps/2048/spec.md`를 읽어봐도 좋다 (문서 형식 참고용, 내용은 무관).

## 요구사항 (사용자 요청)
- 16x16 격자에 도트(픽셀)를 찍어서 그림을 그릴 수 있는 픽셀 아트 에디터.
- 색상 팔레트를 제공해서 원하는 색을 선택해 그릴 수 있어야 한다.
- 완성한 그림을 PNG 파일로 저장(다운로드)할 수 있어야 한다.
- 모바일에서도 사용할 수 있어야 한다 (터치로 도트 찍기).

## 해야 할 일
1. 16x16 그리드 상태를 어떻게 표현하고 렌더링할지 설계한다 (각 셀의 색상 상태 관리).
2. 파일 구조를 정한다. 예시:
   - `/apps/pixel-art-editor/index.html`
   - `/apps/pixel-art-editor/style.css`
   - `/apps/pixel-art-editor/app.js`
   (실제 구성은 네가 판단해서 정해도 된다)
3. UI/UX를 설계한다: 16x16 캔버스(그리드) 영역, 색상 팔레트 UI(기본 제공 색상 + 필요하면 커스텀 색상 선택 `<input type="color">` 등), 지우개 기능, 전체 지우기 버튼, PNG로 저장 버튼.
4. 그리기 동작 설계: 클릭/터치로 한 칸 찍기, 마우스를 누른 채 드래그(또는 터치 드래그)로 여러 칸 연속으로 칠하는 동작까지 지원할지 결정하고 명시한다.
5. PNG 저장 방식을 설계한다: `<canvas>` 엘리먼트에 16x16 픽셀 데이터를 그린 뒤 `canvas.toDataURL('image/png')` 또는 `toBlob`으로 다운로드 링크를 생성하는 방식을 기본으로 고려한다. 실제 화면에 보이는 격자는 확대된 UI이고, 저장용 캔버스는 16x16 원본 해상도로 그리는 방식(또는 배율을 선택할 수 있게 하는 방식)을 검토한다.
6. 접근성/엣지케이스: 빈 캔버스로 저장 시도, 팔레트에서 선택된 색상 표시(현재 선택 색 강조), 다크모드 대응, 모바일 터치 시 스크롤과 그리기 동작이 충돌하지 않도록 하는 방법(`touch-action` 등)을 간단히 언급한다.

## 산출물
- 위 내용을 정리한 계획을 **`/apps/pixel-art-editor/spec.md`** 파일로 작성한다.
- spec.md에는 다음 섹션을 포함한다: 개요, 파일 구조, 데이터 모델(그리드 상태) 설계, UI/UX 설계, 그리기 입력 처리(마우스/터치), 색상 팔레트 설계, PNG 저장 설계, 엣지케이스, 향후 Build 단계에서 참고할 구현 순서.
- spec.md 외에 다른 파일은 생성하거나 수정하지 않는다. 코드를 미리 작성하지 않는다 (필요하면 구조를 보여주는 간단한 의사코드 정도는 허용).
