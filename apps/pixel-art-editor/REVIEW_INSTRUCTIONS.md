# Review 서브에이전트 지침 — 픽셀 아트 에디터

## 역할
너는 이 블로그 프로젝트(/Users/hasuncho/Desktop/my-blog)의 "Review" 단계를 담당하는 서브에이전트다.
Build 서브에이전트가 구현한 `/apps/pixel-art-editor/` 코드를 검증한다. 너는 Build를 수행한
서브에이전트와 다른 세션이며, 독립적으로 검증해야 한다.

## 범위 제한 (반드시 준수)
- 검증 대상은 `/apps/pixel-art-editor/` 폴더 (index.html, style.css, app.js).
- 문제를 발견해 수정할 때도 **`/apps/pixel-art-editor/` 폴더 내부 파일만** 수정한다.
- 블로그 본체 파일(`/index.html`, `/css/`, `/js/`, `/posts/` 등)과 다른 앱(`/apps/2048/` 등)은 건드리지 않는다.
- `spec.md`, `PLAN_INSTRUCTIONS.md`, `BUILD_INSTRUCTIONS.md`는 수정하지 않는다.

## 검증 항목
1. **spec.md 대비 구현 검토**: `/apps/pixel-art-editor/spec.md`에 명시된 요구사항이 실제로
   구현되었는지 확인한다 (16x16 그리드 상태 관리, 클릭/드래그로 칠하기, 기본 팔레트 + 커스텀
   색상, 지우개, 전체 지우기, PNG 저장(16배 확대, 투명 배경 지원), 모바일 터치 지원, 다크모드).
2. **브라우저 실동작 확인**: 로컬 정적 서버(이미 `node scripts/dev-server.js`로 4321 포트에
   떠 있을 수 있으니 먼저 확인하고, 없으면 새로 띄운다)로 `/apps/pixel-art-editor/index.html`을
   열어 다음을 확인한다.
   - 페이지 로드 시 콘솔 에러 없이 16x16 빈 그리드가 보이는지
   - 팔레트에서 색상을 선택하고 그리드를 클릭하면 해당 색으로 칠해지는지
   - 드래그로 여러 칸이 연속으로 칠해지는지
   - 지우개 도구로 칠한 칸을 지울 수 있는지, 전체 지우기 버튼이 동작하는지
   - 커스텀 색상 선택(`<input type="color">`)이 동작하는지
   - PNG 저장 버튼 클릭 시 다운로드가 정상적으로 트리거되는지(에러 없이)
   - 라이트/다크 모드 각각에서 레이아웃이 깨지지 않는지, 그린 픽셀 색상이 테마 전환과 무관하게
     유지되는지
   - 뷰포트를 모바일 크기로 줄였을 때 레이아웃이 정상인지
   브라우저 자동화 도구가 없는 환경이라면, 코드 정적 분석 + 로직 단위 검증(직접 함수 추출
   테스트 등)으로 대체하되 이를 review.md에 명시한다.
3. **코드 품질 점검**: 문법 오류, 명백한 버그(인덱스 계산 오류, 포인터 캡처/드래그 처리 오류,
   이벤트 리스너 중복 등록, 메모리 누수 가능성), 접근성(aria-label, aria-pressed, role 등),
   CLAUDE.md의 기술 제약 위반 여부(프레임워크/번들러 사용, 불필요한 외부 의존성 추가 등)를
   점검한다.
4. 문제를 발견하면 **네가 직접 수정**한다 (범위 내에서). 수정 후 다시 검증한다.

## 산출물
- 검증 결과를 `/apps/pixel-art-editor/review.md`로 작성한다. 포함 내용:
  - 검증 방법 (실제 브라우저 확인 여부, 대체 방법 사용 시 그 이유)
  - 발견한 문제 목록과 심각도
  - 수정한 내용 (있다면)
  - 최종 결론: 배포(Embed 단계 진행) 가능 여부

## 완료 후
review.md 요약과 최종 결론(문제 없음 / 수정 완료 / 추가 조치 필요)을 답변한다.
